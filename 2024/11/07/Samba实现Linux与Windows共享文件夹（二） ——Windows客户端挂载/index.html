Samba实现Linux与Windows共享文件夹（二） ——Windows客户端挂载

https://zhuanlan.zhihu.com/p/164721714

https://blog.csdn.net/qq_44078824/article/details/119847027

第一部分 无报错的情况下的正常操作步骤

一、按键盘Win+R，输入：\\58.49.144.9 然后回车Enter

二 、此时文件夹网络出先share文件夹，双击输入密码

三、选择“记住我的凭据”，再确定

四、右键打开“映射网络驱动器”

五、输入\\58.49.144.9\share即可

六、最终，在网络位置即可出现共享磁盘。

第二部分 有报错如何解决

备注：一般报错都会出现在正常步骤的第二步

一、现象：“你不能访问此共享文件夹,因为你组织的安全策略组织未经身份验证的来宾访问。”

解决：

1、按window+R键输入gpedit.msc 来启动【本地组策略编辑器】→【管理面板】→【网络】→【lanman工作站】→【启用不安全的来宾登录】。

2、将“未配置”设置成“已启用”，再“确定”

二、现象：Windows访问共享文件报错：请检查名称拼写。否则，网络可能有问题

解决：

1、点击键盘Win+R，输入cmd，回车，输入下列命令：

net use \\58.49.144.9\ipc$ “ycwl@2024!” /user:”administrator”

出现“命令成功完成”，即成功。

三、现象：不允许一个用户使用一个以上用户名与服务器或共享资源的多重连接，中断与此服务器或共享。

解决：

1、点击Win+R，打开命令行窗口，输入cmd，回车

2、按照如下命令输入：

net use 回车

net use \\58.49.144.9\ipc$ /delete

显示成功即可。
