Ubuntu20.04安装NFS服务器

参考文档：

前言：

  • NFS 协议默认是不加密的，它不像 Samba，它不提供用户身份鉴别。服务端通过限定客户端的IP地址和端口来限制访问。
  • 服务器和客户端应该能够通过私有局域网互相连接。如果无法提供私有IP地址，你可以使用公有地址，配置服务器的防火墙，并且允许来自可信源的流量通过端口2049。

环境：

服务端Ubuntu20.04IP：10.0.0.10
客户端Ubuntu20.04IP：10.0.0.20

概览：

一、服务端操作

1、服务端安装NFS服务器软件包；

2、服务端设置共享目录给客户端；

3、使服务端配置生效；

4、防火墙配置开放端口；

二、客户端操作

1、客户端安装NFS文件系统的工具；

2、挂载文件系统（临时挂载/永久挂载）

三、测试客户端访问共享是否成功

一、服务端操作

1、服务端安装NFS服务器

安装NFS服务端

sudo apt install nfs-kernel-server

查看支持版本

cat /proc/fs/nfsd/versions

-2 +3 +4 +4.1 +4.2

查看运行情况

systemctl status nfs-server

2、服务端设置共享目录给客户端

vim /etc/exports

——————框内参考———————–

/srv/nfs4 192.168.33.0/24(rw,sync,no_subtree_check,crossmnt,fsid=0)

/srv/nfs4/backups 192.168.33.0/24(ro,sync,no_subtree_check) 192.168.33.3(rw,sync,no_subtree_check)

/srv/nfs4/www 192.168.33.110(rw,sync,no_subtree_check)

/srv/nfs4/data 10.0.0.11(rw,sync,no_subtree_check)

访问权限说明：

  • /srv/nfs4: 这是被共享的目录的路径。
  • 192.168.33.0/24: 允许该子网内的所有主机访问此目录。/24 表示子网掩码为 255.255.255.0，即 192.168.33.0 到 192.168.33.255 范围内的 IP 地址都可以访问。
  • rw: 读写权限，客户端可以读写此目录。
  • ro: 只写权限，客户端可以写此目录。
  • sync: 确保所有的写操作在返回客户端之前都完成，提供数据一致性保障。
  • no_subtree_check: 禁用子树检查。子树检查用于确保目录层级结构的一致性，但在某些情况下（如共享根目录），禁用它可以提升性能。
  • crossmnt: 允许在挂载点跨文件系统进行操作。这通常用于共享挂载点的场景。
  • fsid=0: 指定这个共享目录是根目录，fsid=0 通常用于指定 NFS 根文件系统。

#本次教程的设置

#对客户端IP：10.0.0.20，设置共享文件目录：/home/nfs，只写权限

vim /etc/exports

/home/nfs 10.0.0.20(ro,sync,no_subtree_check,no_all_squash)

补充：若想/home/nfs目录共享的容量是数据盘，则需要在服务端提前进行格式化/挂载

root@localhost:/home# mkfs.ext4 /dev/sdb

root@localhost:/home# mount -t ext4 /dev/sdb /home/nfs

3、使服务端配置生效

生效配置

sudo exportfs -ra

重新导出所有共享目录。这会使 NFS 服务器重新加载 /etc/exports 文件中的配置，并应用任何更改。

sudo exportfs -r

以详细模式输出

sudo exportfs -v

导出所有共享目录，而不仅仅是更新过的目录

sudo exportfs -a

#查看服务器共享信息

showmount -e

#重新加载 NFS 服务器的配置文件，使新的配置生效，而无需停止和启动服务

systemctl reload nfs

4、防火墙配置端口

允许192.168.33.0/24段IP访问NFS服务

sudo ufw allow from 192.168.33.0/24 to any port nfs

// 或 直接放开2049端口

ufw allow 2049

验证

sudo ufw status

二、客户端操作

1、客户端安装NFS文件系统工具

ubuntu安装

sudo apt install nfs-common

centos安装

sudo yum install nfs-utils

2、挂载文件系统（临时挂载/永久挂载）

注意：在 IP 为10.0.0.20的客户端机器上操作。这台机器拥有对/home/nfs目录的读写操作权限。

#临时挂载

sudo mount -t nfs -o vers=4 10.0.0.10:/home/nfs /nfs_bak

注意：/nfs_bak：客户端挂载目录

永久挂载

vim /etc/fstab

10.0.0.10:/srv/nfs4/backups /backups nfs defaults,timeo=900,retrans=5,_netdev 0 0

10.0.0.10:/srv/nfs4/www /srv/www nfs defaults,timeo=900,retrans=5,_netdev 0 0

#验证永久挂载

卸载（此目录为挂载点目录）

umount /backups

umount /srv/www

挂载

mount -a

三、测试客户端访问共享是否成功

#客户端查看磁盘挂载情况

df -Th
